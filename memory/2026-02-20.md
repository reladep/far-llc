# 2026-02-20 ‚Äî Alerts MVP Completed, Workers & APIs Built

## Summary
Completed Alerts MVP build: database schema, ingestion workers, API routes, and frontend component. Set up cron jobs for ongoing data ingestion.

## What Was Built

### Database Schema (004_alerts_mvp.sql)
- `firm_snapshots` ‚Äî historical snapshots of firm data for change detection
- `news_articles` ‚Äî scraped news with relevance scoring
- `firm_alerts` ‚Äî detected changes (fee changes, AUM swings, disclosures)
- `alert_subscriptions` ‚Äî user firm alert subscriptions
- `user_notifications` ‚Äî user-specific alert deliveries
- Custom ENUMs: `alert_type`, `alert_severity`
- RLS policies for all tables

### Workers (in `/far/workers/`)
- `config.js` ‚Äî shared Supabase client loading `.env.local`
- `edgar-rss.js` ‚Äî EDGAR EFTS poller (15 min). Currently returning 0 filings, needs URL refinement
- `adv-diff.js` ‚Äî compares `firmdata_current` vs `firm_snapshots`, batched (20 parallel). Generated 107 fee change alerts, 1,000 snapshots
  - Thresholds: AUM 10%, employees 20%, clients 15%
- `news-scraper.js` ‚Äî Google News RSS per firm with relevance scoring. Scraped ~970/1945 firms: 2,776 articles, ~1,400 news alerts
  - Rate limited to 1 req/sec

### API Routes
- `GET /api/alerts/firm/[crd]` ‚Äî public firm alerts
- `GET /api/alerts/news/[crd]` ‚Äî public firm news
- `GET/PATCH /api/user/alerts` ‚Äî authenticated user notifications
- `GET/POST/DELETE /api/user/alerts/subscriptions` ‚Äî manage subscriptions
- Paywall enforced at API layer (not RLS)

### Frontend Component
- `FirmAlerts.tsx` ‚Äî tabbed Alerts/News UI
- Wired into `/firm/[crd]/page.tsx` below fee calculator
- Scrollable container (400px max), "Show More" button (5 initial, +10 per click), fetches 50 items

### Cron Jobs (isolated sessions, delivery: none)
- `EDGAR RSS Poller` (id: `d6a8130d`) ‚Äî `*/15 * * * *`
- `ADV Diff Checker` (id: `7b52b7ff`) ‚Äî `0 */6 * * *`
- `News Scraper` (id: `c728044f`) ‚Äî `30 * * * *` (hourly, changed from twice-hourly to avoid overlap)

## Key Decisions
- Supabase Dashboard paste for migrations (no CLI)
- Workers as standalone Node scripts (cron-triggered)
- Public read RLS on alerts/news, paywall at API layer
- Free tier = last 7 days, paid = full history
- CRD numbers as primary key for firm matching
